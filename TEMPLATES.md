# SVG Template Contract

Соглашение для дизайнеров и разработчиков. Любой SVG-шаблон, добавляемый в систему, должен следовать этим правилам.

---

## Именование файлов

```
assets/templates/card_v1_{width}x{height}.svg
```

Примеры:
```
assets/templates/card_v1_1080x1350.svg   ← портрет 4:5 (по умолчанию)
assets/templates/card_v1_1080x1080.svg   ← квадрат 1:1
```

**Правила:**
- Единственный шаблон — `card_v1`. Новые варианты создаются только как новые размеры.
- Размер — точно `{width}x{height}` без пробелов.
- Файл должен быть валидным XML/SVG.

---

## Допустимые размеры

| Размер    | Соотношение | Назначение             |
|-----------|-------------|------------------------|
| 1080×1350 | 4:5         | Instagram, ВКонтакте (по умолчанию) |
| 1080×1080 | 1:1         | Квадратный формат      |

## Структура карточки — 4 элемента

```
┌─────────────────────────────┐
│                             │
│   [фото / заливка цветом]   │
│                             │
│                             │
├─────────────────────────────┤
│ ▌ Заголовок                 │  ← обязательный
│   Подзаголовок              │  ← опциональный
│ ─────────────────────────── │
│ [Тег]                       │  ← опциональный
└─────────────────────────────┘
```

## Jinja2-переменные

Шаблон получает следующий контекст. Все переменные всегда присутствуют в рендере.

### Обязательные

| Переменная | Тип    | Описание                              |
|------------|--------|---------------------------------------|
| `title`    | string | Основной заголовок. Макс 120 символов.|
| `palette`  | string | `"light"` или `"dark"`                |

### Опциональные

| Переменная  | Тип           | Описание                                      |
|-------------|---------------|-----------------------------------------------|
| `subtitle`  | string        | Подзаголовок / лид. Пустая строка `""` если не задан. |
| `tag`       | string        | Короткий тег: `"AI"`, `"Новости"`, `"Tech"`. Пустая строка `""` если не задан. |
| `bg_color`  | string / None | CSS hex `#RRGGBB`. `None` если фоновое фото (Pillow вставит изображение). |

### Цветовые переменные (устанавливаются внутри шаблона)

Шаблон сам задаёт цвета через `{% set %}` на основе `palette`. Не передавай их снаружи.

| Переменная   | Light          | Dark      | Назначение          |
|--------------|----------------|-----------|---------------------|
| `c_bg`       | `#F5F5F0`      | `#1A1A2E` | Фон контентной панели |
| `c_title`    | `#111111`      | `#FFFFFF` | Цвет заголовка      |
| `c_sub`      | `#555555`      | `#AAAACC` | Цвет подзаголовка   |
| `c_tag_bg`   | `#E8F0FE`      | `#2E2E4E` | Фон тега            |
| `c_tag_fg`   | `#1A73E8`      | `#E8C547` | Текст тега          |
| `c_accent`   | `#1A1A2E`      | `#E8C547` | Акцентная линия, разделитель |

---

## Режимы фона

| Условие          | Поведение                                       |
|------------------|-------------------------------------------------|
| `bg_color = "#RRGGBB"` | Сплошная заливка. Контентная панель использует `c_bg`. |
| `bg_color = None`      | Фоновое фото (Pillow compositing). SVG рендерится с прозрачным фоном и затемняющим оверлеем. |

### Использование в шаблоне

```svg
<!-- Сплошная заливка -->
{% if bg_color %}
<rect width="1080" height="1350" fill="{{ bg_color }}"/>
{% endif %}

<!-- Затемняющий оверлей поверх фото -->
{% if not bg_color %}
<rect width="1080" height="1350" fill="rgba(0,0,0,0.45)"/>
{% endif %}

<!-- Контентная панель: разный цвет для двух режимов -->
{% if bg_color %}
<rect x="0" y="820" width="1080" height="530" fill="{{ c_bg }}"/>
{% else %}
<rect x="0" y="820" width="1080" height="530" fill="rgba(0,0,0,0.6)"/>
{% endif %}
```

## Правила именования слоёв в Figma

При экспорте SVG из Figma имена слоёв становятся атрибутами `id` элементов.

### Рекомендуемые id

| id            | Элемент  | Назначение                                     |
|---------------|----------|------------------------------------------------|
| `bg`          | `<rect>` | Фоновый прямоугольник (заменить на Jinja2-блок) |
| `overlay`     | `<rect>` | Затемняющий оверлей поверх фото                |
| `content-panel` | `<rect>` | Контентная панель снизу                      |
| `accent-line` | `<rect>` | Вертикальная акцентная линия                   |
| `divider`     | `<rect>` | Горизонтальный разделитель                     |
| `title`       | `<text>` | Основной заголовок → заменить на `{{ title }}` |
| `subtitle`    | `<text>` | Подзаголовок → обернуть в `{% if subtitle %}`  |
| `tag-bg`      | `<rect>` | Фон тега → обернуть в `{% if tag %}`           |
| `tag-label`   | `<text>` | Текст тега → заменить на `{{ tag }}`           |

### Процесс конвертации Figma → Jinja2

1. Экспортируй фрейм как SVG (File → Export → SVG).
2. Открой SVG в редакторе.
3. В начало файла добавь блок `{% if palette == "dark" %}...{% else %}...{% endif %}` с цветовыми переменными.
4. Замени статичные цвета на переменные: `fill="..."` → `fill="{{ c_title }}"` и т.д.
5. Замени текстовое содержимое на Jinja2-переменные: `{{ title | truncate(38, True, '…') }}`.
6. Условные элементы (subtitle, tag) оберни в `{% if %} ... {% endif %}`.
7. Для фона добавь двойной блок `{% if bg_color %}` / `{% if not bg_color %}`.
8. Положи файл в `assets/templates/card_v1_{width}x{height}.svg`.

## Скелет нового шаблона

```svg
<svg xmlns="http://www.w3.org/2000/svg"
     width="1080" height="1350" viewBox="0 0 1080 1350">

  <!--
    card_v1_1080x1350.svg

    Jinja2-переменные:
      title    (str, обязательный)
      subtitle (str, '' если не задан)
      tag      (str, '' если не задан)
      palette  ("light" | "dark")
      bg_color (CSS hex | None)
  -->

  <!-- Цвета по палитре -->
  {% if palette == "dark" %}
    {% set c_bg     = "#1A1A2E" %}
    {% set c_title  = "#FFFFFF" %}
    {% set c_sub    = "#AAAACC" %}
    {% set c_tag_bg = "#2E2E4E" %}
    {% set c_tag_fg = "#E8C547" %}
    {% set c_accent = "#E8C547" %}
  {% else %}
    {% set c_bg     = "#F5F5F0" %}
    {% set c_title  = "#111111" %}
    {% set c_sub    = "#555555" %}
    {% set c_tag_bg = "#E8F0FE" %}
    {% set c_tag_fg = "#1A73E8" %}
    {% set c_accent = "#1A1A2E" %}
  {% endif %}

  <!-- 1. Фон -->
  {% if bg_color %}
  <rect width="1080" height="1350" fill="{{ bg_color }}"/>
  {% endif %}

  {% if not bg_color %}
  <rect width="1080" height="1350" fill="rgba(0,0,0,0.45)"/>
  {% endif %}

  <!-- Контентная панель снизу -->
  {% if bg_color %}
  <rect x="0" y="820" width="1080" height="530" fill="{{ c_bg }}"/>
  {% else %}
  <rect x="0" y="820" width="1080" height="530" fill="rgba(0,0,0,0.6)"/>
  {% endif %}

  <!-- Акцентная линия -->
  <rect x="64" y="840" width="6" height="72" rx="3" fill="{{ c_accent }}"/>

  <!-- 2. Заголовок -->
  <text x="90" y="892"
        font-family="'Inter', 'Arial', sans-serif"
        font-size="52" font-weight="700"
        fill="{{ c_title }}"
        dominant-baseline="middle">
    {{ title | truncate(38, True, '…') }}
  </text>

  <!-- 3. Подзаголовок -->
  {% if subtitle %}
  <text x="90" y="972"
        font-family="'Inter', 'Arial', sans-serif"
        font-size="30" font-weight="400"
        fill="{{ c_sub }}"
        dominant-baseline="middle">
    {{ subtitle | truncate(55, True, '…') }}
  </text>
  {% endif %}

  <!-- Разделитель -->
  <rect x="64" y="1030" width="952" height="1" fill="{{ c_accent }}" opacity="0.25"/>

  <!-- 4. Тег -->
  {% if tag %}
  <rect x="64" y="1055"
        width="{{ (tag | length) * 14 + 40 }}" height="40"
        rx="20" fill="{{ c_tag_bg }}"/>
  <text x="84" y="1075"
        font-family="'Inter', 'Arial', sans-serif"
        font-size="18" font-weight="600"
        fill="{{ c_tag_fg }}"
        dominant-baseline="middle">
    {{ tag }}
  </text>
  {% endif %}

</svg>
```

## Добавление нового размера

1. Создай SVG по скелету выше, скорректировав координаты под новый размер.
2. Положи в `assets/templates/card_v1_{width}x{height}.svg`.
3. Добавь кортеж `(width, height)` в `ALLOWED_SIZES` в `app/services/card_spec.py`.
4. Добавь тест в `render_test.py` и кейс в `figma_test.py`.
5. Обнови `ParsedCardRequest.size` Literal в `app/llm/parser.py`.

> LLM-парсер и системный промпт **не требуют изменений** — все варианты используют одни и те же Jinja2-переменные.
